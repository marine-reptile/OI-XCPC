#### QOJ 10040

##### 题目来源

[Petrozavodsk Summer 2024. Day 5: Der Kontest E题](https://qoj.ac/contest/1902/problem/10040)

##### 题目描述

有一棵树，树上有 $n$ 个点，每条边有长度。从第 $i$ 个点走到第 $j$ 个点的代价是 $a_i+dist(i,j)b_i$。其中 $a_i,b_i$ 是 $i$ 的两个属性，$dist(i,j)$ 表示 $i,j$ 在树上的最短路径的长度。

现在我们从 $1$ 点开始，希望对 $2$ 到 $n$ 的每个点 $i$，求出从 $1$ 走到 $i$ 的最小代价。

##### 输入格式

第一行一个整数 $n$，表示树的点数。

第二行 $n$ 个整数，第 $i$ 个表示第 $i$ 个点的 $a_i$。

第三行 $n$ 个整数，第 $i$ 个表示第 $i$ 个点的 $b_i$。

接下来 $n-1$ 行每行三个整数 $(u,v,l)$，表示树上一条 $(u,v)$ 的边，长度为 $l$。

##### 输出格式

一行 $n-1$ 个整数，第 $i$ 个表示从 $1$ 走到点 $i+1$ 的最小代价。

##### 输入输出样例 #1

###### 输入 #1

```
3
0 1 2
8 4 4
1 2 1
1 3 7
```

###### 输出 #1

```
8 41
```

##### 输入输出样例 #2

###### 输入 #2

```
2
353 313
928248 475634
2 1 898027
```

###### 输出 #2

```
833591767049
```

##### 说明/提示

$2\le n\le 10^5$，$0\le a_i\le 10^{12}$，$1\le b_i,l\le 10^6$

##### 题目解析

我们考虑点分树，然后在点分树的每一层上把所有点按照到中心的距离排序。

然后，我们尝试通过类似 dijkstra 算法的思路，每次确定一个点的答案，然后将它作为一个新的支点去更新所有其他的点。发现我们可以在点分树上在每一层的分治子树中更新其他点。

考虑，假设我们确定了一个点 $x$ 的答案是 $ans_x$，那么我们枚举它的所有点分树祖先 $y$，设 $x$ 到 $y$ 的距离为 $dis(x,y)$。那么我们的操作相当于，对于当前分治子树的所有点 $z$，将它的答案对 $a_x+ans_x+b_x(dis(x,y)+dis(y,z))$ 取 $\min$。那么发现，其中与 $z$ 有关的只有一项 $dis(y,z)$，可以以它为变量，维护所有可能的 $x$ 在当前层 $y$ 形成的一次函数。

发现，我们不需要每次处理所有的 $z$，因为 $dis(y,z)$ 在当前层子树内是单调的——我们只关心所有目前还没有找到答案的点中距离当前分治中心最近的那个。

因此，我们将每一层分治子树内的所有点按照到分治中心的距离排序，对每一层分治子树，我们只关心当前最近的还没有被找到答案的点。发现这一过程不需要考虑两点的树上最短路是否真的经过分治中心，因为如果不经过，那么在下一层分治中心再处理一定会比在这一层更优。

所以，我们的算法结构如下：

1. 对每一层分治子树，维护一个凸壳，表示对所有当前子树内已经确定的 $x$，它的答案将以什么形式贡献给当前分治子树内的其他点。此部分使用李超线段树实现。

2. 我们使用一个堆，维护目前所有的分治子树中，下一个将被确定答案的点中答案最小的，每次我们将其拿出并确定这个点的答案。

3. 当我们确定一个点的答案，就在它的所有分治子树祖先上将它代表的直线加入凸壳。
4. 如果我们当前分治子树内的下一个点已经被其他子树确定，就将其弹出，然后将下一个点放入堆。

复杂度瓶颈是在所有点分树祖先上加入一个直线，复杂度 $O(n\log^2 n)$。如果我们使用动态李超树维护凸壳，那么每次加入直线只会有 $O(1)$ 的空间开销，因此空间复杂度 $O(n\log n)$。
