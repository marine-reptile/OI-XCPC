### 历史版本和

考虑有序列 $a_1,a_2,\cdots ,a_n$，维护两种操作：

+ 区间加 $v$。
+ 求在过去的所有版本中，$\sum_{i=l}^ra_i$ 的和。

#### 线段树历史版本和（区间加区间查询）

考虑对线段树的每个节点维护 $sum,sum_h,tag,tag_h,tag_t$，分别表示当前区间的和，当前区间的历史和，累计在当前区间的区间加标记，累计在当前区间的区间历史和加标记，累计在当前区间的版本更新标记。

考虑下传标记的流程：

```cpp
inline void addtag(int i,int v,ll h,int t){
    sg[i].hs+=t*sg[i].s+h*(sg[i].r-sg[i].l+1);
    sg[i].s+=v*(sg[i].r-sg[i].l+1);
    sg[i].th+=t*sg[i].tg+h;
    sg[i].tg+=v;
    sg[i].t+=t;
}
```

发现第一步做的是处理当前新下传的版本更新标记，用 $sum$ 更新 $sum_h$，旧的 $tag$ 更新旧的 $tag_h$。

然后 $tag$ 和 $tag_h$ 分别更新 $sum$ 和 $sum_h$，也即对应的信息互相更新。

对 $sum,sum_h,len$ 写成向量的形式：
$$
\begin{pmatrix}sum&sum_h&len\end{pmatrix}\begin{pmatrix}1&0&tag\\tag_t&1& tag_h\\0&0&1\end{pmatrix}=\begin{pmatrix}sum'&sum_h'&len'\end{pmatrix}
$$

#### CF 1824 D - LuoTianyi and the Function

考虑在第 $i$ 个位置，维护所有数最后一次出现的位置集合，设这个集合为 $S$，那么 $g(x,i)$ 就是 $S$ 中第一个 $\ge x$ 的数。

发现每次要将 $i$ 的答案置为 $i$，并找到 $a_i$ 上一次出现的位置 $d$，设其前驱和后继分别是 $l$ 和 $r$，那么实际上就是本来 $[l+1,x]$ 这一段区间的答案是 $x$，现在修改成 $r$，区间加上 $r-x$ 就好了。

发现修改操作形如 $[l,r]*[i,\infty]$ 矩形加的形式，查询操作形如 $[l,r]*[x,y]$ 的形式，考虑先离线，转化为 $[l,r]*[0,y]$ 和 $[l,r]*[0,x-1]$ 的差，这可以直接使用历史版本和解决。

时间复杂度 $O(n\log n)$，常数略大。



#### NOIP2022 T4 - 比赛

我们发现，我们可以逐位处理，对于 $a$ 和 $b$ 分别维护单调栈，每次相当于弹出单调栈 $\le a_i$ 的段，置为 $a_i$，也可以转化为对这一段置为 $a_i-a_j$（$j$ 是被弹出的，原来掌管这一段的）。我们发现，因为弹栈的次数是 $O(n)$，所以总操作数是 $O(n)$ 的。

那么就是要维护：对两个序列 $\{a\}$ $\{b\}$，每次支持对 $a$ 或 $b$ 区间加，查询 $\sum_{i=l}^r a_ib_i$ 的历史和。

我们考虑在线段树上累计 $a$ 的历史贡献 $ta$、$b$ 的历史贡献 $tb$，$ab$ 的历史贡献 $tc$ 以及原来的内容的历史贡献 $t$。

考虑下传 $tag$，我们发现，需要记录 $sa$ 表示 $a$ 的区间和，$sb$ 表示 $b$ 的区间和，$sc$ 表示 $c$ 的区间和，以及历史和。

每次对历史和的贡献就是 $sc$ 的时间贡献，$ta$ 和 $sb$ 的贡献，$tb$ 和 $sa$ 的贡献以及 $tc$ 的贡献。

考虑合并 $tag$，只需要将原来的 $tag$ 在 $t$ 的时间内产生的贡献以及新加入的 $ta$ 和 $tb$（这是 $t$ 的时间之后产生的贡献）和原来的 $tag$ 产生的贡献计入 $tc$ 就可以了。

```cpp
inline void node::addtag(ull A,ull B,ull T,ull HA,ull HB,ull HC){
    ull len=r-l+1;
    sh+=T*sc+HA*sb+HB*sa+HC*len;
    sc+=A*sb+B*sa+A*B*len;
    sa+=A*len,sb+=B*len;

    hc+=T*ta*tb+HA*tb+HB*ta+HC;
    ha+=T*ta+HA,hb+=T*tb+HB;
    ta+=A,tb+=B,t+=T;
}
```

  

#### CF 997E - 

首先找到这题的弱化版，526F。526F是固定对 $[1,n]$ 查询的版本。

我们的方法是，既然需要 $mx-mn=r-l$，我们利用单调栈和线段树区间加维护前面所有数到当前位置区间的 $mx-mn+l$。发现 $mx-mn\ge r-l$，所以 $r$ 是最小值。又因为对 $[r,r]$ $mx=mn$，所以最小值 $r$ 一定能达到。

那么就转化成区间加全局查询最小值个数，我们维护线段树节点的最小值和最小值个数，合并的时候讨论一下即可。

然后考虑加强成区间加区间查询最小值个数，发现也是容易的，返回答案的时候也一样讨论合并一下就可以了。

最后考虑加上历史和，我们发现，如果一个点被打上了历史和 $tag$，那么在它上一次被访问和这一次被访问之间的区间最小值个数是不变的，改变的只会是最小值。

所以我们在 $[l,r]$ 下传历史和标记的时候，只对左右存在 $[l,r]$ 最小值的部分下传，它们就是答案。所以存在历史和标记的部分也必定是在这部分版本存在全局最小值的部分（否则历史和标记不会从第一层下传到现在）。

那么就做完了。

```cpp
inline void addtag(int i,int x,ll t){
    sg[i].mn+=x,sg[i].tg+=x;
    sg[i].ans+=t*sg[i].res;
    sg[i].t+=t;
}
inline void pushdown(int i){
    int x=sg[i<<1].mn,y=sg[i<<1|1].mn;
    addtag(i<<1,sg[i].tg,x<=y?sg[i].t:0);
    addtag(i<<1|1,sg[i].tg,y<=x?sg[i].t:0);
    sg[i].tg=0,sg[i].t=0;
}
inline void pushup(int i){
    sg[i].ans=sg[i<<1].ans+sg[i<<1|1].ans;
    sg[i].mn=min(sg[i<<1].mn,sg[i<<1|1].mn);
    sg[i].res=0;
    if(sg[i].mn==sg[i<<1].mn)sg[i].res+=sg[i<<1].res;
    if(sg[i].mn==sg[i<<1|1].mn)sg[i].res+=sg[i<<1|1].res;
}
```
